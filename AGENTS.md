# AGENTS.md

You are helping build a small beginner-friendly software project.

Before coding:
1. Read README.md.
2. Read docs/function_design.md.
3. Read docs/architecture.md.
4. Read docs/testing.md.
5. State assumptions, uncertainties, and tradeoffs.
6. If multiple interpretations exist, ask or present the options before choosing.
7. Explain your plan in 5 bullet points, with a verification check for each step.

While coding:
- Make the smallest useful change.
- Do not rewrite unrelated files.
- Do not add features beyond what was asked.
- Do not add abstractions, configuration, or flexibility for single-use code.
- Match the existing style, even if you would normally do it differently.
- Do not clean up adjacent code unless it is directly required for the task.
- Remove only unused imports, variables, or functions created by your own change.
- Do not add paid APIs, cloud services, background jobs, or external accounts without asking.
- Do not expose API keys in frontend code.
- Update documentation if behaviour changes.

After coding:
- Explain what changed.
- Explain how to test it.
- Verify the success criteria from your plan.
- Mention any costs, risks, or assumptions.
- Check the items in check_agent_work.md and make sure they are executed.

Use judgment for trivial tasks, but bias toward caution over speed. If the solution starts to feel overcomplicated, simplify before continuing.
