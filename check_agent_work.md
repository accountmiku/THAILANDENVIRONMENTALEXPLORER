# Check Agent Work

## Before accepting the agent's work

- [ ] Did the agent read the relevant project files before coding?
- [ ] Did the agent state assumptions, uncertainties, and tradeoffs before coding?
- [ ] Did the agent ask or present options when the task had multiple interpretations?
- [ ] Did the agent follow `AGENTS.md`?
- [ ] Did the agent follow `docs/function_design.md`?
- [ ] Did the agent preserve the architecture in `docs/architecture.md`?
- [ ] Did the agent respect the domain assumptions in `docs/domain.md`?
- [ ] Did the agent follow the coding style in `docs/style.md`?
- [ ] Did every changed line trace directly to the task?
- [ ] Did the agent avoid speculative features, unnecessary abstractions, and unused configurability?
- [ ] Did the agent add or update tests where needed?
- [ ] Did the agent define and verify clear success criteria?
- [ ] Did the app still run after the change?
- [ ] Did the agent explain what it changed?
- [ ] Did the agent avoid unnecessary changes outside the task?
- [ ] Did the agent save important new decisions into project `.md` files rather than leaving them only in chat?
- [ ] Is there any important context missing from the files that future AI agents would need?

## Final question

If another AI agent opened this project tomorrow, would it understand what we are building and why?
