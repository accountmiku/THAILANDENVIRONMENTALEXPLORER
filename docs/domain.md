# Domain Notes

## Purpose

Capture domain-related knowledge that the AI may not know. This file can describe scientific assumptions, game rules, business rules, data rules, or any other project-specific domain knowledge.


## Example: Water Quality Domain Notes

- pH is usually between 0 and 14.
- Freshwater pH is often around 6.5 to 8.5, but this varies by context.
- Turbidity should not be negative.
- Temperature should be recorded in degrees Celsius.
- The app should not make scientific claims without user confirmation.

## Reasonable Value Ranges

To be filled in for the chosen app.

## Missing Value Rules

- Every displayed area must identify its data sources and latest update date.
- Missing source metadata must not be presented as verified environmental data.

## Scientific Cautions

- Current area summaries and scores are fictionalized prototype data.
- The prototype is updated manually and is not connected to live environmental datasets.
- Future real datasets must include source names, links where available, and dataset dates before their values are displayed.
