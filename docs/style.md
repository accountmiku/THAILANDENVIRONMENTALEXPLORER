# Style Guide

## Purpose

Keep code and interface style consistent.

## Coding Style

- Prefer simple, readable code over clever code.
- Use descriptive names.
- Add comments only where they help beginner-level readers.
- Keep changes small and focused.
- Avoid rewriting unrelated files.
- Do not add features beyond what was requested.
- Do not create abstractions for single-use code.
- Do not add configurability or flexibility unless the current task needs it.
- Match the existing style in nearby files.
- Mention unrelated dead code or cleanup opportunities instead of changing them.

Every changed line should trace directly to the user's request. If a solution becomes much longer than the problem requires, simplify it before continuing.

## UI Style

- Keep pages visually clean and uncluttered.
- Use clear labels for inputs and buttons.
- Show validation errors close to the relevant input.
- Prefer predictable layouts over decorative complexity.
